Font Testing in Chromium Browsers

If you're seeing this page, you've been asked to help out testing some font display issues, specifically ones that appear in Chromium browsers such as Google Chrome. But testing in other browsers is very much appreciated!

Why this is important: Braille Toolbox uses a common font feature, ligatures, to display properly formatted braille signs. However, browsers seem to treat the font feature differently.

Test protocol: All tests have a simple verification target, that the word "braille" should appear as the three-cell contraction "brl", not seven individual characters. No testing is needed for screen readers at this point.

Testing Steps

Test these in sequence without refreshing the page (start again if you refresh). Do not use uppercase letters.

Test 1
Without modifying the field contents, look at the placeholder text. It should appear properly contracted.
Pass
Test 2
Type the word "braille" in the field. The new text should replace the placeholder field and contract properly. However this step appears to fail for Chromium browsers.
Pass
Test 3
Next, clear the field. Then copy the text braille and then paste back into the field. Even for Chromium browsers, this should contract properly.
Pass
Test 4
Lastly, type the word "braille" again after your pasted version, it should contract properly. For some reason, after pasting, contraction works properly even when typing new content. But please report a failure if contraction is not working.
Pass

Please report your results as a general bug, entering your test results in the issue details field.

Please use the format "Pass, Fail, Pass, Pass" along with any other context. Thank you!
